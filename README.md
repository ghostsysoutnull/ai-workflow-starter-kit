# Gemini-CLI Software Project Starter Kit

A docs-first starter kit for building software projects with Gemini CLI and agent-driven workflows.

## What this repository contains

This kit is designed for three audiences:

- **Project builders** who want a repeatable way to plan and ship work with Gemini CLI
- **Agent authors** who need reusable role definitions, prompt structures, and handoff patterns
- **Team maintainers** who want shared conventions, templates, and operating guidelines

## Start here

1. Read [docs/overview.md](docs/overview.md)
2. Follow [docs/getting-started.md](docs/getting-started.md)
3. Pick an agent pattern from [docs/agents/README.md](docs/agents/README.md)
4. Choose a workflow from [docs/strategies/README.md](docs/strategies/README.md)
5. Reuse templates from [docs/templates/agent-brief-template.md](docs/templates/agent-brief-template.md)
6. Adapt prompts from [docs/prompts/README.md](docs/prompts/README.md)

## Content principles

This starter kit favors:

- practical guidance over abstract theory
- examples that can be copied and adapted
- explicit handoffs between planning, research, implementation, and review
- small reusable documents rather than one giant playbook

## First-pass scope included here

- agent role documents
- workflow strategy guides
- configuration patterns
- starter templates
- best-practice references
- prompt examples

Use this repository as a base, then tailor the docs to your own team, stack, and delivery constraints.
