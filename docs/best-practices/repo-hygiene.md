# Repository Hygiene

A clean repo makes agent-assisted work faster and more accurate.

## Recommended habits

- keep docs near the work they explain or link them clearly from a central index
- preserve file naming patterns once chosen
- avoid dumping long-lived notes into random locations
- update docs when workflows or conventions change
- prefer one source of truth for a process

## Good starter structure

A docs-first repository can group content by intent:

- agents
- strategies
- configs
- templates
- best-practices
- prompts

## Maintenance routine

Review periodically for:

- stale docs
- duplicate guidance
- conflicting conventions
- prompts that no longer match actual workflow

## Anti-patterns

- creating dozens of docs with no index
- storing critical conventions only in chat history
- keeping examples that are too generic to reuse
- letting templates drift from how the team really works
