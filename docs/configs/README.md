# Configuration Guidance

This section covers practical configuration patterns for running Gemini CLI in a consistent way across software projects.

## What these docs cover

- how to define project conventions
- how to describe agent operating modes
- how to keep prompts and outputs aligned with team standards
- how to document defaults when no formal machine-readable config exists yet

## Files in this section

- [agent-config-patterns.md](agent-config-patterns.md)
- [project-conventions.md](project-conventions.md)

## Principle

A lightweight documented convention is better than an undocumented habit.

If your project does not yet have a fixed JSON, YAML, or TOML schema for agent setup, start by writing the rules in markdown and convert them into a formal config later.
