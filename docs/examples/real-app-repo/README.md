# Real App Repo Markdown Pack

This folder is a sample markdown pack for a realistic software repository using Gemini CLI.

## Intended use

Copy these files into a real project when you want a ready-made operating layer for agent-assisted delivery.

## Included files

- [repo-overview.md](repo-overview.md)
- [project-brief.md](project-brief.md)
- [conventions.md](conventions.md)
- [agent-roster.md](agent-roster.md)
- [current-priorities.md](current-priorities.md)
- [release-checklist.md](release-checklist.md)
- [handoff-example.md](handoff-example.md)

## Example target repo shape

This pack assumes a normal application repository with areas such as:

- `src/`
- `tests/`
- `docs/`
- CI configuration
- environment or deployment files

## How to adopt

1. copy the files into a `docs/project/` or `.project/` folder in your app repo
2. replace placeholder project details
3. trim sections you do not need
4. keep one owner for each file so it stays current
