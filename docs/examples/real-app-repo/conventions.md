# Project Conventions

## Task handling

- start with the user or business outcome
- use research first when the relevant code path is unclear
- require a plan for multi-file or risky work
- keep diffs focused on the approved task

## Validation

- run targeted tests when code changes
- document manual verification for UI behavior
- explicitly note skipped validation and why
- update docs when workflows or behavior change

## Documentation

- keep overview docs short and linked
- prefer examples over abstract explanations
- remove stale instructions instead of stacking warnings on top of them

## Collaboration

- use explicit handoffs between discovery, planning, implementation, and review
- separate blockers from optional improvements
- surface assumptions early
