# Release Checklist

Use this checklist before a feature or fix is considered ready to release.

## Scope and behavior

- requested outcome is satisfied
- non-goals remained out of scope
- compatibility concerns were reviewed

## Validation

- relevant automated checks passed
- required manual checks were completed
- skipped checks are recorded with rationale

## Documentation

- user-facing behavior changes are documented
- contributor workflow changes are documented
- obsolete guidance was removed or updated

## Risk

- rollback or mitigation notes exist for risky changes
- known residual risks are called out
- ownership for follow-up items is clear
