## How to use the examples

1. copy the closest prompt
2. replace placeholders with project facts
3. add constraints and validation rules
4. ask for a structured output
5. save the prompt if it works well

## Prompt quality checklist

A strong prompt usually includes:

- role or task framing
- target outcome
- scope boundaries
- constraints
- output format
- validation or success checks
