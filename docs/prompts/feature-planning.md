# Feature Planning Prompts

## Prompt: bounded implementation plan

> Act as a Planner Agent. Convert this feature request into a phased implementation plan. Include assumptions, likely files or subsystems, risks, validation checkpoints, and any open questions. Do not write code.

## Prompt: acceptance criteria review

> Review this feature request and improve the acceptance criteria. Identify vague requirements, likely edge cases, and missing validation expectations. Keep the output structured and concise.

## Prompt: thin-slice proposal

> Propose the smallest useful first slice for this feature. Explain why it is a good first milestone, what it intentionally excludes, and how success should be validated.
