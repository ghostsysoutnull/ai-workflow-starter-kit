# Project Setup Prompts

Use these prompts at the start of a new repo or workstream.

## Prompt: initial repository assessment

> Inspect this workspace as a Gemini CLI project starter. Identify the current structure, missing foundation docs, and the smallest useful set of starter materials to create first. Return findings, gaps, and a short implementation plan.

Why it works:

- asks for discovery before action
- requests prioritization
- asks for a bounded artifact

## Prompt: kickoff conventions

> Help me define operating conventions for this project. Create a concise markdown convention sheet covering task framing, scope control, validation expectations, documentation updates, and handoff rules. Keep it practical and easy to adopt.

## Prompt: starter doc map

> Create a docs-first information architecture for this Gemini CLI project. Organize content for agent roles, strategy guides, config guidance, best practices, and prompt examples. Keep the structure simple enough for a small team to maintain.
