# Strategy Guides

Strategy guides describe repeatable ways to run common software-project tasks with Gemini CLI.

## Available guides

- [project-kickoff.md](project-kickoff.md) — establish scope, repo context, and work lanes
- [feature-delivery.md](feature-delivery.md) — move from requirement to implementation safely
- [debugging.md](debugging.md) — isolate defects and verify fixes
- [refactoring.md](refactoring.md) — improve code without changing intended behavior
- [documentation-workflows.md](documentation-workflows.md) — produce and maintain useful project docs

## How to choose

Use this simple rule:

- Unknown codebase or new initiative: `project-kickoff`
- New capability or change request: `feature-delivery`
- Broken behavior or failing tests: `debugging`
- Cleanup and maintainability work: `refactoring`
- Missing or outdated docs: `documentation-workflows`

## Common pattern across all strategies

1. clarify the target outcome
2. identify constraints and risks
3. gather the minimum necessary context
4. produce a bounded plan
5. execute in small verifiable steps
6. validate with evidence
7. document the result
