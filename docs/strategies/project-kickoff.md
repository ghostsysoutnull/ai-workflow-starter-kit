# Project Kickoff Strategy

Use this strategy when a project, repository, or major workstream needs structure before execution begins.

## Goal

Move from a broad idea to a working plan with clear constraints, roles, and next actions.

## Sequence

### 1. Capture the initiative
Record:

- business or user outcome
- in-scope deliverables
- out-of-scope items
- hard constraints
- success signals

### 2. Run discovery
Use the `Research Agent` to map:

- repo structure
- important docs
- existing patterns
- risk areas
- open unknowns

### 3. Create the first plan
Use the `Planner Agent` to turn the research into:

- phases
- milestones
- dependencies
- validation checkpoints

### 4. Define operating conventions
Document project defaults such as:

- branch strategy
- testing expectations
- documentation rules
- prompt style conventions
- review requirements

### 5. Start with a thin slice
Do not begin with the largest task. Pick one useful, bounded piece that proves the workflow.

## Deliverables

A good kickoff usually produces:

- a project brief
- a first-phase plan
- a list of working conventions
- one starter prompt for each major role

## Risks to manage

- hidden assumptions about stack or architecture
- no agreement on validation
- oversized first milestone
- no saved conventions after initial success
